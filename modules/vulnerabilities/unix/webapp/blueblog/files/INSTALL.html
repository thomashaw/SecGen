BlueBlog installation and configuration

Congratulations on downloading BlueBlog weblog application!

This document will get you ready to start blogging within minutes. But first, we'll need to make sure you have the other components needed by BlueBlog.

Necessary components

To get BlueBlog running in its simplest form, you will need to have downloaded and installed a Java 2 Standard Environment (version 1.4 or later) and a Servlet Engine like Jakarta Tomcat, Resin, Jetty or other. This document will only cover working with Tomcat. Please refer to the servlet engine documentation to install BlueBlog with other servlet engines.

If you already have a recent J2SE and Tomcat up and running, you can skip to the Add Context section below.

Add a web application context for BlueBlog in Tomcat

Application contexts in Tomcat are defined in the server.xml file located in the conf directory under the Jakarta Tomcat root directory. Open this file in a text editor like Wordpad (in Windows) or Emacs (in Unix/Linux). Find the first <Context element, inside of a <Host element. You'll need to add a new Context for BlueBlog, and you may want to remove any contexts that are already there, especially if they have the same context path that you want to use for BlueBlog. (For instance – by default Tomcat installs an /examples context.)

Add a new Context for BlueBlog by adding this text: 

<Context path="/blueblog" docBase="c:\blueblog\web" debug="1" reloadable="true"/>

The entry above assumed that BlueBlog was installed in the c:\blueblog directory. Change this entry to whereever you installed the files. Also, the entry installed BlueBlog in the context path /blueblog. You can change this to whatever suits you. If you want to install BlueBlog as the default (or "root") context, leave the path attribute empty, like this: 

<Context path="" docBase="c:\blueblog\web" debug="1" reloadable="true"/>

If you don't have any other web applications installed on the same server, the above is the recommended way of installing BlueBlog!
